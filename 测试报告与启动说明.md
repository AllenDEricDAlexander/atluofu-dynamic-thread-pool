## 二、启动说明

### 前置条件

1. **Java 环境**：JDK 17+
2. **Redis 服务**：确保 Redis 运行在 `127.0.0.1:6379`
3. **Node.js**：用于前端开发（已安装）

### 1. 配置 Redis 连接

如果 Redis 需要密码，请修改以下配置文件：

**Admin 模块配置** (`dynamic-thread-pool-admin/src/main/resources/application-dev.yml`):
```yaml
redis:
  sdk:
    config:
      host: 127.0.0.1
      port: 6379
      password: YOUR_REDIS_PASSWORD  # 填写你的 Redis 密码
```

**Test 模块配置** (`atluofu-dynamic-thread-pool-test/src/main/resources/application-dev.yml`):
```yaml
dynamic:
  thread:
    pool:
      config:
        host: 127.0.0.1
        port: 6379
        # password: YOUR_REDIS_PASSWORD  # 如果需要密码，取消注释并填写
```

### 2. 启动后端服务

#### 启动 Admin 管理后台（端口 8089）
```bash
cd /Users/mario/SelfProject/be/atluofu-dynamic-thread-pool/dynamic-thread-pool-admin
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```

#### 启动 Test 应用（端口 8093，包含线程池示例）
```bash
cd /Users/mario/SelfProject/be/atluofu-dynamic-thread-pool/atluofu-dynamic-thread-pool-test
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```

### 3. 启动前端服务

```bash
cd /Users/mario/SelfProject/be/atluofu-dynamic-thread-pool/atluofu-dynamic-thread-pool-ui
npm run dev
```

前端服务启动后访问：**http://localhost:3000**

## 五、功能测试

### 1. 查看线程池列表
- 前端页面会自动加载线程池列表
- 点击"自动刷新"按钮，每 3 秒自动更新数据

### 2. 修改线程池配置
1. 点击线程池列表中的"编辑"按钮
2. 修改核心线程数或最大线程数
3. 点击"确认修改"
4. 观察线程池配置是否更新

### 3. 验证动态调整

动态线程池，上报线程池配置：{...}

## 七、常见问题

### 1. Redis 连接失败
**错误**：`NOAUTH Authentication required`
**解决**：在配置文件中设置正确的 Redis 密码

### 2. 端口被占用
**错误**：`Port 8089/8093/3000 is already in use`
**解决**：修改对应服务的端口配置或关闭占用端口的进程

### 3. 前端无法连接后端
**解决**：检查 `vite.config.js` 中的代理配置是否正确
